# Монитор билетов БЖД

Локальный сервис для проверки появления мест на `pass.rw.by`. Пользователь добавляет маршрут или номер поезда, дату и время окончания мониторинга. Сервер проверяет страницу каждые 3 секунды и отправляет Telegram-уведомления не чаще одного раза в минуту.

## Запуск

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env` по примеру `.env.example`:

```bash
PORT=4174
CHECK_INTERVAL_MS=3000
TELEGRAM_MIN_INTERVAL_MS=60000
TELEGRAM_BOT_TOKEN=токен_бота
TELEGRAM_CHAT_ID=chat_id
```

3. Запустить сервис:

```bash
npm run dev
```

Панель откроется на `http://localhost:5173`, API работает на `http://localhost:4174`.

## Telegram

Создайте бота через `@BotFather`, добавьте токен в `.env`, затем узнайте `chat_id` удобным способом: напишите боту любое сообщение и откройте `https://api.telegram.org/bot<токен>/getUpdates`. После этого нажмите в панели кнопку `Проверить Telegram`.

## Проверка билетов

По умолчанию сервис строит ссылку поиска на `pass.rw.by` из маршрута, даты и номера поезда. Если сайт изменит параметры или для конкретного маршрута нужна точная страница результатов, откройте поиск вручную в браузере и вставьте готовую ссылку в поле `Точная ссылка поиска`.

Данные задач хранятся локально в `data/watch-tasks.json`. Файл не коммитится, чтобы не сохранять личные маршруты.
