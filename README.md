# Монитор билетов БЖД

Локальный и серверный сервис для проверки появления мест на `pass.rw.by`. Пользователь добавляет маршрут или номер поезда, дату и время окончания мониторинга. В текущем тестовом режиме сервер выполняет две проверки с интервалом 20 секунд.

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env` по примеру `.env.example`:

```bash
PORT=4174
CHECK_INTERVAL_MS=20000
MAX_CHECKS_PER_TASK=2
TELEGRAM_MIN_INTERVAL_MS=60000
TELEGRAM_BOT_TOKEN=токен_бота
TELEGRAM_CHAT_ID=chat_id
```

3. Запустить сервис:

```bash
npm run dev
```

Панель откроется на `http://localhost:5173`, API работает на `http://localhost:4174`.

## Продакшн-запуск

Сервис умеет работать одним Node-процессом: после сборки сервер отдает и API, и веб-панель.

```bash
npm ci
npm run build
npm start
```

После этого приложение доступно на `http://localhost:4174`.

## Деплой не с домашнего компьютера

GitHub Pages не подходит для основного режима: там нельзя держать постоянный сервер и фоновый таймер. Нужен хостинг для Node-приложения: Render, Railway, Fly.io, VPS или Docker-хостинг.

Для Render уже добавлен `render.yaml`:

1. Загрузите проект в GitHub-репозиторий.
2. В Render создайте `New` -> `Blueprint` и выберите репозиторий.
3. В переменных окружения задайте `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `CHECK_INTERVAL_MS=20000`, `MAX_CHECKS_PER_TASK=2`, `TELEGRAM_MIN_INTERVAL_MS=60000`.
4. После деплоя откройте публичный URL Render. На нем будет та же панель.

Для Docker:

```bash
docker build -t rw-ticket-watch .
docker run -p 4174:4174 --env-file .env rw-ticket-watch
```

## Telegram

Чтобы узнать `TELEGRAM_CHAT_ID`, напишите боту любое сообщение, затем откройте:

```text
https://api.telegram.org/bot<токен>/getUpdates
```

В ответе нужен `message.chat.id`. После настройки нажмите в панели кнопку `Проверить Telegram`.

## Проверка билетов

По умолчанию сервис строит ссылку поиска на `pass.rw.by` из маршрута, даты и номера поезда. Если сайт изменит параметры или для конкретного маршрута нужна точная страница результатов, откройте поиск вручную в браузере и вставьте готовую ссылку в поле `Точная ссылка поиска`.

Данные задач хранятся локально в `data/watch-tasks.json`. Файл не коммитится, чтобы не сохранять личные маршруты.
